我的计算机网络-应用层笔记

应用层

网络应用就是计算机网络存在的理由。本章学习有关计算机网络应用原理和实现方面的知识。
几个关键的应用层概念：

  • 应用层协议原理
  • Web和HTTP
  • FTP文件传输协议
  • 电子邮件的传输协议SMTP
  • 因特网的目录服务器DNS
  • P2P应用
  • TCP套接字

应用层协议原理

研发网络应用程序的核心是写出能够运行在不同的端系统和通过网络彼此通信的程序。而这个通信过程你不用去关心其他层的，只需要考虑应用层，这种将应用软件限制在端系统的方法，也促进了大量网络程序的迅速开发和部署。

网络应用体系结构

两种主流体系结构：

  • 客户-服务器体系结构(C-S)
  • 对等(P2P)体系结构

客户-服务器体系结构 有一个总是打开的主机称为服务器，服务于来自客户端主机的请求。客户之间是不直接通信的，因为服务器具有固定的,周知的IP地址，并且服务器总是打开的，所以客户可以通过向服务器的IP发送分组与其联系。常见的基于客户-服务器体系结构的应用程序：Web,FTP,Telent,电子邮件。

P2P体系结构 对位于数据中心的专用服务器有最小的或者没有依赖。应用程序在间断连续的的主机对之间使用直接通信，这些主机对被称为 对等方。P2P最重要的特性之一就是它的 自扩展性 ，又因为它通常不需要庞大的服务器基础设施和服务器带宽，所以它也是成本有效的。
P2P应用面临的三个主要挑战：

  • ISP友好。多数住宅ISP受制于非对称的带宽应用，下载比上载多得多。而P2P需要上载流量。
  • 安全性。这是因为P2P的高度分布 和 开放 特性
  • 激励。使用户自愿的向应用提供带宽，存储和计算资源

进程通信

操作系统术语中，进行通信的实际上是 进程 而不是程序。一个进程可以被认为是运行在端系统中的一个程序。
两个不同端系统上的进程通过跨越计算机网络交换 报文 而互相通信。具体过程就是，发送进程生成并向网络发送报文，接收进程接收报文并将报文发送回去进行相应。

可供应用程序使用的运输服务

应用程序对运输服务的要求：

  • 可靠数据传输：为了避免分组在计算机网络中丢失，确保应用程序发送的数据正确安全的交付而提供一个确保数据交付的服务，就认为提供了 可靠数据传输
  • 吞吐量：发送进程能够向接收进程交付比特的速率。
  • 定时：交互式实时应用程序会有要求，时延。
  • 安全性：运输协议能进行数据的加密-解密，以防数据在两个进程之间被观察到。

因特网提供的运输服务

因特网为应用程序提供了两个运输层协议：UDP TCP
| 应用 | 数据丢失 | 带宽 | 时间敏感|
| ——– | :—–: | :—-: | :—-: |
| 文件传输 | 不能丢失 | 弹性 | 不 |
| 电子邮件 | 不能丢失 | 弹性 | 不 |
| Web文档 | 不能丢失 | 弹性 | 不 |
| 因特网电话/视频会议| 不能丢失 | | |
1.TCP
TCP服务模型包括面向连接服务和可靠数据传输服务。当应用程序调用TCP作为其运输协议时，应用程序就能获得来自TCP的这两种服务。TCP协议还具有 拥塞控制机制 （见第3章）

  • 面向连接的服务：在应用层数据报文开始流动之前，TCP让客户和服务器互相交换运输层控制信息。这个所谓的握手过程提示客户和服务器，使他们为大量分组的到来做好准备。在握手阶段后，一个 TCP连接 就在两个进程的套接字之间建立了。这条连接是全双工的，即连接双方的进程可以在此链接上同时进行报文收发。当应用程序结束报文发送时，必须拆除该链接。
  • 可靠的数据传输服务通信进程能够依靠TCP，无差错，按适当顺序交付所有发送的数据。当应用程序的一端将字节流传进套接字时，它能够依靠TCP将相同的字节流交付给接收方的套接字，而没有字节的丢失和冗余。
    2.UDP
    UDP是一种不提供不必要服务的轻量级运输协议，它仅提供最小服务。UDP是无法连接的，因此在两个进程通信之前没有握手过程。UDP协议提供一种不可靠数据传输服务，当进程将一个报文送进UDP套接字时，UDP协议并不保证该报文将到达接收进程，到达接收进程的报文也可能是乱序到达的。UDP协议没有 拥塞控制机制
    3.因特网运输协议不提供的服务
